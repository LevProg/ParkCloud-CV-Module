# ParkCloud CV Module — Детекция занятости парковочных мест

## Обзор

Модуль компьютерного зрения для платформы ParkCloud, обеспечивающий анализ занятости парковочных мест в реальном времени на основе видеопотоков с камер наблюдения.

### Возможности

- **Калибровка камер**: Восстановление перспективы по реперным точкам (гомография) — точное отображение между координатами карты объекта и пикселями камеры
- **Детекция автомобилей**: YOLOv8 для обнаружения транспортных средств (car, bus, truck)
- **Анализ занятости**: Определение занятости каждого машиноместа (0–100%) на основе пересечения проекции места и обнаруженного ТС
- **Мультикамерное слияние**: Комбинирование показаний нескольких камер по одному месту (4 стратегии)
- **Поток событий**: SSE (Server-Sent Events) для передачи событий в реальном времени
- **Веб-интерфейс**: Интерактивная карта парковки с визуализацией состояния

## Быстрый старт

### Установка

```bash
# Клонировать репозиторий
cd parkcloud-cv

# Создать виртуальное окружение
python -m venv venv
venv\Scripts\activate          # Windows

# Установить зависимости
pip install -r requirements.txt
```

### Запуск (демо-режим)

```bash
# Запуск с демо-режимом (синтетические данные, без камер)
uvicorn src.main:app --host 0.0.0.0 --port 8000 --reload

# Открыть в браузере: http://localhost:8000
```

### Запуск (продакшен)

```bash
# С YOLO-детектором
USE_YOLO=true YOLO_MODEL=yolov8n.pt DEMO_MODE=false \
  uvicorn src.main:app --host 0.0.0.0 --port 8000

# С GPU
DEVICE=cuda USE_YOLO=true DEMO_MODE=false \
  uvicorn src.main:app --host 0.0.0.0 --port 8000
```

## API

### REST Endpoints

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/` | Веб-интерфейс |
| GET | `/api/status` | Сводка по парковке |
| GET | `/api/spaces` | Все места + статус |
| GET | `/api/spaces/{id}` | Статус конкретного места |
| GET | `/api/cameras` | Информация о камерах |
| GET | `/api/map` | Полная карта объекта |
| GET | `/api/events` | Последние события |
| GET | `/api/stream` | **SSE** поток событий |
| POST | `/api/process_frame/{cam_id}` | Обработка кадра |
| POST | `/api/process_frame/{cam_id}/annotated` | Кадр с аннотацией |
| POST | `/api/config/fusion?strategy=...` | Сменить стратегию слияния |
| GET | `/api/config` | Текущая конфигурация |

### Формат событий (SSE)

```json
{
  "event_type": "space_occupied",
  "space_id": "A05",
  "timestamp": "2026-03-04T12:00:00Z",
  "occupancy_pct": 85.3,
  "confidence": 0.92,
  "camera_id": "cam_01",
  "previous_pct": 12.0
}
```

### Типы событий

| Тип | Описание |
|-----|----------|
| `space_occupied` | Место стало занятым |
| `space_freed` | Место освободилось |
| `occupancy_update` | Обновление % занятости |
| `full_snapshot` | Полный снимок состояния |

## Векторная карта (parking_map.json)

Карта объекта содержит:

- **Координатную систему** (метрическая, origin = левый верхний угол)
- **Парковочные места** — полигоны в мировых координатах
- **Камеры** — с реперными точками для калибровки, рабочей зоной и списком покрываемых мест
- **Полосы движения** — для визуализации

Каждая камера калибруется по 4+ реперным точкам (соответствия «карта → пиксель»), из которых вычисляется матрица гомографии.

## Стратегии слияния (Multi-Camera Fusion)

| Стратегия | Описание |
|-----------|----------|
| `weighted_average` | Средневзвешенное по confidence |
| `max_confidence` | Использовать камеру с наибольшей уверенностью |
| `max_occupancy` | Максимальный % занятости среди камер |
| `vote` | Голосование: занято, если >50% камер подтверждают |

## Конфигурация через env-переменные

| Переменная | По умолчанию | Описание |
|-----------|-------------|----------|
| `PARKING_MAP` | `config/parking_map.json` | Путь к карте |
| `USE_YOLO` | `true` | Использовать YOLO |
| `YOLO_MODEL` | `yolov8n.pt` | Путь к модели |
| `DEVICE` | `cpu` | Устройство (`cpu` / `cuda`) |
| `FUSION_STRATEGY` | `weighted_average` | Стратегия слияния |
| `DEMO_MODE` | `true` | Демо-режим |
